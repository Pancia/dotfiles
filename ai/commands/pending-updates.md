# Process Pending CLAUDE.md Updates

Find and apply pending CLAUDE.md update suggestions from `.cc/pending-updates-*.md` files.

## Procedure

1. List all `.cc/pending-updates-*.md` files in the current directory. If none exist, say "No pending updates found." and stop.

2. Read the current `CLAUDE.md` file. If it doesn't exist, warn the user and stop.

3. Process each pending update file **one at a time**, oldest first:
   a. Read the file and summarize what it suggests (one line)
   b. Determine the edit to make to CLAUDE.md -- the file format varies (Replace/With blocks, "add after X" instructions, etc.) so interpret the intent rather than parsing mechanically
   c. Check if a previous update in this batch already made the same or overlapping change -- if so, note it's already covered and skip to cleanup
   d. Apply the edit to CLAUDE.md using the Edit tool (the user will approve/reject via normal permissions)
   e. If the user approved the edit, delete the pending update file
   f. If the user rejected the edit, leave the file in place and move on

4. After processing all files, report how many were applied and how many were skipped/rejected.

5. If any updates were applied, commit the changes (only CLAUDE.md and deleted pending-update files — do not commit unrelated working copy changes):
   a. Detect the project's VCS: check for `.jj/` (Jujutsu) or `.git/` (Git). Refer to CLAUDE.md for project-specific VCS instructions if available.
   b. For **jj** repos: `jj split -r @ CLAUDE.md .cc/ -m "<message>"` (splits only the relevant paths into a new commit, leaving other working copy changes untouched)
   c. For **git** repos: stage only CLAUDE.md and any deleted `.cc/pending-updates-*` files, then commit
   d. Commit message format: subject line `Apply <N> pending CLAUDE.md update(s)`, followed by a body that briefly describes what was added or changed

## Important

- Read CLAUDE.md fresh before each edit (it changes after each applied update)
- The pending update files are generated by Haiku and the format is not standardized -- some use `Replace:`/`With:` blocks, some say "add this after [section]", some use `**Current:**`/`**Replacement:**`. Understand the intent and apply it.
- When the suggested replacement text doesn't exactly match what's in CLAUDE.md (Haiku sometimes gets whitespace or minor details wrong), find the closest matching section and apply the intended change.
- Do not combine multiple pending updates into a single edit -- apply them individually so the user can approve each one.
